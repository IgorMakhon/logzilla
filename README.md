# logzilla
# Logs Analyzer
# Для чего нужен:
# Существует сервис, в котором часто происходят ошибки, но не на все из них нужно мгновенно реагировать. 
# Многие ошибки обрабатываются автоматически. Например, при падении доступа к базе во время выполнения запроса возникает 
# ошибка, но сразу после этого происходит переподключение и запрос выполняется снова. Необходимость почитать логи этого 
# приложения возникает только когда с приложением что-то идет не так и нужно найти ошибки, указывающие на то, в чем 
# проблема. Логи огромные, разделены на несколько файлов, каждая ошибка занимает очень много строк. 
# Что-то искать глазами в логах практически нереально. Поэтому нужно сделать программу, которая сможет прочитать 
# лог-файл или все лог-файлы из папки и выбрать логи, которые соответствуют заданным критериям:

# Дата:
# → равная указанной
# → больше указанной
# → меньше указанной
# → находится в заданном диапазоне
# Текст:
# → Поиск по наличию текста в сообщении
# → Поиск по отсутствию текста в сообщении (отсеивание ненужных)
# Поиск должен производиться для каждого сообщения целиком, не для отдельных строк. 
# В самом начале анализа, строки, по логике являющиеся частью блока нужно объединить в блоки (цельные сообщения)
# Результат:
# Вывести сообщения, соответствующие заданным критериям
# Так как каждое сообщение очень длинное, по умолчанию выводить первые 300 символов сообщения. 
# Создать возможность вывести сообщения целиком
# Если производился поиск по тексту (не по отсутствию), то выводить 150 символов до найденного текста, сам текст и 
# 150 символов после текста
# Дополнительные, нефункциональные требования:
# Программой смогут пользоваться другие
# Программу будет несложно поддерживать самому или кому-то другому
# Программу не стыдно выложить в свое портфолио

# Этапы:
# распознать аргументы, которые ввел пользователь
# определить что указал пользователь: файл или папку
# получить содержимое файла (файлов)
# разбить содержимое на блоки и сохранить блоки в переменной
# реализовать поиск по дате
# реализовать поиск по тексту
# настроить удобный вывод результатов
